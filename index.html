Featured Project

Sshell project (Command-Line)

This is a team college project that took over 700 lines of code. Our goal was to provide a friendly command line tool that can manage simple builtin commands such as cd, pwd, exit, |, exit and handle input mistakes. My task was to ‚Äúimplement a pipeline command.‚Äù Made an struct array called command_line which was utilized in each function throughout the code. Parsing is done by getting user‚Äôs input and breaking it into parts to then store the information to the corresponding structs. In execution, the sshell will fork and create a child. That child will continue to fork and create children such that each child runs one command in pipeline. Therefore, it is the grandchildren of sshell running the commands. The child will fork and pipe in a loop manner. In the loop, it will create a pipe, fork, and the grandchildren will connect stdin and stdout to pipes. The first grandchild and the last grandchildren are treated differently, for they will still read partially from stdin and stdout. The child will wait for all grandchildren to terminate and collect exit status. It will process exit status into a string and pass that string to shell by a pipe resulting in the execution of all commands in the pipeline to get bundled together. More details in my GitHub.üôÇ
